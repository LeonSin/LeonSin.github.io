Digest of [Django vs Flask vs Pyramid: Choosing a Python Web Framework] 10月 17 2014

简介

　　在众多 python web 框架中，本文只关注Flask, Pyramid和Django。

  • Flask是一个基于小规模应用，实现简单需求的“微框架”
  • Pyramid旨在开发大型WEB应用，开发者可根据项目的实际情况，自行决定选择合适的（外部）工具进行扩展。
  • Django同样用于开大型WEB应用，本身包含一整套工具包，开发者只需要引入即可进行扩展
  • 另外，Django实现了一个ORM（Pyramid和Flask则没有）；应用最广的非Django系应用的ORM框架是SQLAlchemy。

发展

  • 最年轻的框架是 Flask，始于2010年中段；但已有大量有影响力的小规模WEB应用建立在其上，比如httpbin，一个用来调试/测试HTTP库的实用工具。
  • Pyramid 起源于Pylons project(首版发布于2005年)，在2010年后端被命名为 Pyramid。
  • Django 首版发布于2006年。

社区

　　Django在StackOverflow有超过80000个问题。虽然Flask和Pyramid没有如此规模，但社区活跃，在Github上，stars数量几乎相同
　　他们都是BSD-derived permissive licenses。

对Bootstrap的支持

　　Django和Pyramid均有内建的Bootstrap工具；Flask则没有，因为它的目的并不是为了创建大型的MVC应用。

总结

　　Pyramid在三者之中是最复杂的，他可创建的应用既大可小。
　　最广泛使用的是Django。
