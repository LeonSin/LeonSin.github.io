notes_on_Flask 12月 9 2014

Why Flask

  1. 文档。Flask的官方文档齐全，初学者易入手
  2. 微内核，与Django这种全栈式框架不同，Flask保持内核的精简，同时提供良好的扩展机制。
  3. 其源码本身是很好的Python学习材料

What is Flask

Flask是一个微内核的Web开发框架。
Flask主要依赖两个库：

  1. Jinja2。模板引擎
  2. Werkzeug。 WSGI套件

“微内核”的定义：

The “micro” in microframework means Flask aims to keep the core simple but extensible.

内核精简，易于扩展，功能不失强大，比如，一般的Web应用都需要和数据库交互。某些框架中，Django已经集成了ORM框架，而Flask不会束缚你，你可以使用SQLAlchemy,也可以使用MongoEngine。在开源社区中，有大量优秀的Flask扩展，主要有：

  1. Flask-WTF。基于WTForms的表单验证扩展。
  2. flask-mongoengine。MongoEngine（Document-Object Mapper，类似关系数据库中的ORM）的Flask扩展，方便我们操作MongoDB。同时集成WTForm，统一数据库和表单的验证。
  3. Flask-Script。为Flask应用提供编写脚本的功能。可以用来运行一个开发服务器，也可以与数据库交互。

Digest of [Django vs Flask vs Pyramid: Choosing a Python Web Framework] 10月 17 2014

简介

　　在众多 python web 框架中，本文只关注Flask, Pyramid和Django。

  • Flask是一个基于小规模应用，实现简单需求的“微框架”
  • Pyramid旨在开发大型WEB应用，开发者可根据项目的实际情况，自行决定选择合适的（外部）工具进行扩展。
  • Django同样用于开大型WEB应用，本身包含一整套工具包，开发者只需要引入即可进行扩展
  • 另外，Django实现了一个ORM（Pyramid和Flask则没有）；应用最广的非Django系应用的ORM框架是SQLAlchemy。

发展

  • 最年轻的框架是 Flask，始于2010年中段；但已有大量有影响力的小规模WEB应用建立在其上，比如httpbin，一个用来调试/测试HTTP库的实用工具。
  • Pyramid 起源于Pylons project(首版发布于2005年)，在2010年后端被命名为 Pyramid。
  • Django 首版发布于2006年。

社区

　　Django在StackOverflow有超过80000个问题。虽然Flask和Pyramid没有如此规模，但社区活跃，在Github上，stars数量几乎相同
　　他们都是BSD-derived permissive licenses。

对Bootstrap的支持

　　Django和Pyramid均有内建的Bootstrap工具；Flask则没有，因为它的目的并不是为了创建大型的MVC应用。
